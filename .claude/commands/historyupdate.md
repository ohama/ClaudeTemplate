# historyupdate

HISTORY.md에 주요 이벤트를 기록합니다.

## 절차

### 1. 현재 세션 분석
- 이번 세션에서 발생한 주요 이벤트 파악
- 다음 항목에 해당하는 것만 기록:
  - **Decision**: 중요한 기술적 결정
  - **Change**: 구조적 변경, 대규모 리팩토링
  - **Issue**: 발견된 문제, 버그, 기술 부채

### 2. HISTORY.md 업데이트
```markdown
## YYYY-MM-DD
- Decision: [결정 내용]
- Change: [변경 내용]
- Issue: [이슈 내용]
```

### 3. 판단 기준

**기록해야 하는 것**:
- 아키텍처 변경
- 새로운 의존성 추가
- 중요한 버그 수정
- 기술 스택 변경

**기록하지 않는 것**:
- 일반적인 기능 구현
- 작은 버그 수정
- 코드 정리

## 규칙

- 간결하게 (1-2문장)
- 날짜별로 그룹화
- 중복 기록 방지
