# tdd

TDD (Red → Green → Refactor) 사이클을 실행합니다.

## 절차

### 1. Red (실패하는 테스트 작성)
- 구현하려는 기능에 대한 테스트를 먼저 작성
- 테스트 실행하여 테스트가 **실패**하는지 확인
- 실패 메시지가 예상대로인지 확인

### 2. Green (최소한의 구현)
- 테스트를 통과시키는 **최소한의** 코드 작성
- 완벽한 코드가 아니어도 됨
- 테스트 실행하여 테스트 **통과** 확인

### 3. Refactor (리팩토링)
- 코드 품질 개선 (중복 제거, 명확한 이름 등)
- 테스트는 계속 통과해야 함
- 테스트 실행하여 여전히 통과 확인

## 규칙

- 테스트 없이 프로덕션 코드 작성 금지
- 한 번에 하나의 테스트만 작성
- 버그 수정 시 재현 테스트부터 작성
- 리팩토링 중에는 새 기능 추가 금지

## 완료 후

- .claude/STATE.md 업데이트 (Evidence에 테스트 결과)
- .claude/CACHE.md에 발견 사항 기록 (필요시)
