# howto

현재 작업을 신입개발자용 튜토리얼로 문서화합니다.

## 사용법

```
/howto                    # 현재 작업을 문서화
/howto "주제"             # 특정 주제로 문서 작성
```

## 절차

### 1. 기존 문서 확인
- `docs/howto/` 디렉토리 확인
- 중복 방지를 위해 기존 주제 파악

### 2. 현재 작업 분석
- 완료한 Step 또는 진행 중인 작업 파악
- 여러 독립적 주제가 있으면 분리

### 3. 문서 작성
- `docs/howto/<topic>.md` 생성
- 템플릿에 맞춰 작성

### 4. README 업데이트
- `docs/howto/README.md`에 새 문서 추가

## 문서 템플릿

```markdown
# [주제]: 초보자 가이드

## 개요
- 이 문서에서 배우는 것
- 왜 필요한가

## 사전 준비
- 필요한 도구
- 관련 파일

## 구현
### 1. ...
### 2. ...

## 테스트
```bash
dotnet test
```

## 문제 해결
| 문제 | 해결 방법 |
|------|-----------|
```

## 규칙

- 한 문서는 한 주제
- 코드 예제 필수
- 실행/검증 방법 포함
