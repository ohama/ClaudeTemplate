# Build Issues - 자동 이슈 기록

빌드/테스트 에러 발생 시 Claude가 자동으로 이슈를 기록합니다.

## 활성화 조건

- 빌드 명령어 실행 후 에러 발생 시
- 테스트 실행 후 실패 시
- 컴파일 에러 발생 시

## 자동 기록 규칙

### 에러 발생 시 동작

```
빌드/테스트 실행
    ↓
에러 발생 감지
    ↓
해결 시도 (최대 2-3회)
    ↓
┌─────────────────┬─────────────────┐
│    해결됨 ✅     │    미해결 ❌     │
├─────────────────┼─────────────────┤
│ resolved/에     │ open/에 생성    │
│ 자동 기록       │ 자동 기록       │
│ (해결방법 포함) │ (시도한 방법)   │
└─────────────────┴─────────────────┘
```

### 기록 시점

| 상황 | 동작 |
|------|------|
| 에러 해결 후 | `resolved/`에 해결 방법과 함께 기록 |
| 해결 실패 시 | `open/`에 시도한 방법과 함께 기록 |
| 사용자가 직접 해결 | 기록 여부 질문 |

### 기록하지 않는 경우

- 단순 타이포 수정으로 즉시 해결된 경우
- 이미 동일한 이슈가 기록된 경우
- 사용자가 기록 안함을 선택한 경우

## 기록 내용

자동 기록 시 다음 내용을 포함:

```markdown
# Issue NNNN: [에러 요약]

**Status**: Resolved | Open
**Type**: Build
**Created**: YYYY-MM-DD
**Resolved**: YYYY-MM-DD

## 에러 메시지

```
(전체 에러 출력)
```

## 환경

- OS: (자동 감지)
- 명령어: (실행한 명령어)

## 원인

(분석한 원인)

## Resolution

(해결 방법 - 해결 시)

## 시도한 방법

(미해결 시 시도한 내용)
```

## 사용자 알림

에러 기록 후 사용자에게 알림:

```
빌드 에러가 발생하여 이슈로 기록했습니다.
- 파일: docs/issues/resolved/0003-missing-dependency.md
- 원인: package.json에 의존성 누락
- 해결: npm install lodash 실행
```

## 관련 명령어

- `/issue build` - 수동으로 빌드 이슈 기록
- `/issue list` - 기록된 이슈 목록 확인
