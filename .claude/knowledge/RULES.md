# RULES

프로젝트 개발 규칙입니다.

## TDD 규칙

- 테스트 먼저 작성
- 버그 수정은 재현 테스트부터
- 리팩토링 전 테스트 통과 확인

## 코드 스타일

(프로젝트 코드 스타일 규칙)

## 커밋 규칙

```
<type>: <subject>

type:
- feat: 새 기능
- fix: 버그 수정
- refactor: 리팩토링
- docs: 문서
- test: 테스트
- chore: 기타
```

## 브랜치 규칙

- `main`: 안정 버전
- `feature/*`: 기능 개발
- `fix/*`: 버그 수정

## 금지 사항

- 실패하는 테스트 커밋
- 주석 처리된 코드 커밋
