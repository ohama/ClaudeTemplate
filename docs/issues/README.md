# Issues

프로젝트에서 발생한 이슈와 해결 내용을 기록합니다.

## 목적

- 발생한 문제와 해결 방법 기록
- 동일 이슈 재발 시 빠른 참조
- 기술 부채 및 개선 사항 추적

## 이슈 목록

| 번호 | 제목 | 상태 | 생성일 |
|------|------|------|--------|
| - | - | - | - |

## 사용법

```
/issue                    # 새 이슈 기록
/issue resolve <번호>     # 이슈 해결 기록
```

## 파일 명명 규칙

`NNNN-<slug>.md`

- `NNNN`: 4자리 번호 (0001, 0002, ...)
- `<slug>`: 짧은 설명 (kebab-case)
